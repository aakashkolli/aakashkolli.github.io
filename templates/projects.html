Fault-Tolerant Stream Processing Engine

Distributed stream processing framework for analyzing high-volume event streams with accurate event-time semantics. Supports tumbling, sliding, and session windows while handling out-of-order data, allowing analytics pipelines to produce correct results even when events arrive late or partitions temporarily stall.

Built in Kotlin with Apache Kafka and RocksDB. Implements watermark-based event-time processing, idle-partition detection, Chandy-Lamport distributed snapshots, and transactional Kafka sinks to provide exactly-once delivery guarantees and fault-tolerant state recovery.

Lattice

Collaborative Markdown editor with live previews, comments, cursor presence, and conflict-free editing across multiple users. Comments remain attached to the correct text even as collaborators edit simultaneously, allowing teams to review and discuss documents without losing context.

Built with Next.js, Rust, and Yjs CRDTs. Uses an actor-per-room backend architecture with a custom binary WebSocket protocol, deterministic CRDT synchronization, and server-side update replay that guarantees convergence for reconnecting and late-joining clients.

Rust Next.js Yjs WebSockets

Distributed DAG Task Scheduler

Crash-recoverable workflow orchestration system that executes dependency-based DAGs across distributed workers. Submit a directed acyclic graph of tasks, monitor execution in real time, and recover automatically from worker or coordinator failures without losing workflow state.

Built in Go with gRPC streaming and BoltDB-backed persistence. Uses atomic dependency resolution, heartbeat-based failure detection, UUID execution tokens to reject stale results, and crash recovery logic that reconstructs workflow state and resumes execution after coordinator restarts.

Go gRPC BoltDB Prometheus

Optimal Execution Algorithm Library

Order execution library that slices large equity trades into optimally timed child orders using TWAP, VWAP, and Implementation Shortfall strategies, built on the Almgren-Chriss market impact model. Adaptive IS variant recalibrates trading trajectories at each slice using a constrained BVP solver with RK4 integration, minimizing execution cost and risk under live market conditions.

Built in C++17 with a lock-free order state machine, SPSC ring buffers, and a zero-allocation FIX 4.4 parser with O(1) tag dispatch. Includes a GBM-driven price-time priority matching engine for simulation, Python bindings via pybind11, and a Polars-based post-trade analysis layer.

C++17 Python Eigen

NASDAQ ITCH Order Book Replay System

High-performance market replay engine that reconstructs the full NASDAQ limit order book from raw ITCH feeds. Supports deterministic replay, queue-position fill modeling, and event-level backtesting, enabling realistic simulation of execution behavior and trading strategies.

Built in C++ with zero-copy parsing, custom memory allocators, and Robin Hood hashing. Uses mmap-backed data ingestion, fixed-point arithmetic, and strict price-time priority matching to process millions of market events per second while producing bitwise-reproducible results.

C++17 pybind11 Google Benchmark
